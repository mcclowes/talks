# Why docs as code?

When your docs live in the same repo as your code, you get:

- **Version control** — history, diffs, blame, rollback
- **Review workflows** — PRs, approvals, comments
- **Automation** — CI/CD runs linters, link checkers, deploys
- **Single source of truth** — docs ship with the code they describe
- **Great tooling** - automatic compatibility with AI, custom tools
