# What is docs-as-code?

> Documentation as Code (Docs as Code) refers to a philosophy that you should be writing documentation with the same tools as code: Issue Trackers. Version Control (Git) Plain Text Markup (Markdown, reStructuredText, Asciidoc)

- Write markdown files
- Save/collaborate with Git(Hub)
- Publish with a website generator

---

# Why docs-as-code?

When your docs live in the same repo as your code, you get:

- **Version control** — history, diffs, blame, rollback
- **Review workflows** — PRs, approvals, comments
- **Automation** — CI/CD runs linters, link checkers, deploys
- **Single source of truth** — docs ship with the code they describe
- **Great tooling** - automatic compatibility with AI, custom tools

It's easier than ever with AI help.
