# Why Claude?

**Tooling ecosystem** — Claude Code, Skills, MCP, system prompts. Built for workflows, not just chat.

**Strong at writing** — Consistently ranks highest on long-form and nuanced text. Docs aren't code — tone, structure, and clarity matter.

**Large context window** — Up to 200k tokens. Feed it an entire docs site and it keeps track.

---

# Anthropic: a more responsible provider

**Constitutional AI** — Claude is trained to be helpful, harmless, and honest. Not just RLHF — a published set of principles.

**Transparency** — Anthropic publishes safety research, model cards, and usage policies. You can read the reasoning.

**Cautious by default** — Claude asks before acting, flags uncertainty, and avoids hallucinating with confidence.

**Enterprise-ready** — Data retention controls, SOC 2, no training on your data by default. Matters when docs contain internal context.

# Good at...

*(out of the box)*

- Drafting from a rough brief
- Reformatting existing content
- Suggesting structure for complex docs
- Explaining technical concepts simply
- Summarising long changelogs

---

# Not good at...

*(out of the box)*

- Knowing your product
- Maintaining your voice (by default)
- Catching its own factual errors
- Knowing what's changed since training
- Working across multiple files

# Projects

### Giving Claude persistent context

{{image-right:/images/wtd-london-docs-that-think/claude-projects.png|Claude projects UI}}

- **Files** — upload style guides, API specs, glossaries. Claude references them in every conversation.
- **Project instructions** — a custom prompt that shapes every response. Your tone, your rules, your constraints.
- **Shared knowledge** — everyone on the project gets the same context without copy-pasting.

This is where Claude.ai stops being generic and starts knowing your product.
