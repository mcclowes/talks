Systematically work through each GitHub issue in this repository and address them in order, starting with a recent high-priority issue, or just the most recent issue if none are marked high-priority.

Ignore:
- Closed issues
- Assigned issues

Steps:
1. Fetch open, unassigned issues from this repository
2. Prioritize by labels (e.g., "priority: high", "urgent", "critical") if present
3. If no high-priority issues exist, pick the most recently created issue
4. Read the issue details and understand the requirements
5. Implement the solution following the project's coding standards
6. Create a PR that references the issue with "Fixes #X"

Then repeat until there are no issues to tackle!
